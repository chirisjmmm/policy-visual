# 정책 효과 예측 시각화 (Policy Visual)

BK21·세종과학펠로우십 정책 문서를 입력으로, LLM·멀티에이전트가 단계별 토의
(forward→backward)를 거쳐 정책 효과를 예측하는 연구의 결과를 웹에서 보여주는
정적(static) 시각화입니다. 빌드 도구·외부 라이브러리 없이 순수 HTML/CSS/JS로
동작합니다.

## 실행

```bash
cd web
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

GitHub Pages 등 정적 호스팅에 `web/` 폴더를 그대로 올려도 동작합니다.

## 구성 — 3개 탭

### ① 이해관계 구조 (Knowledge Graph)
- 러프한 원본 KG(`kr_bk21_policy_graph.json`, `kr_ssf_policy_graph.json`)를 그대로
  쓰지 않고, **정책 문서(md)를 기준으로 재점검·재구성**한 그래프입니다.
  중복 엣지를 통합하고, 누락된 핵심 관계를 보강했습니다.
- 관계(엣지)는 **종류별로 색을 구분**(자금지원/관리·위탁/평가·심의/선정/지원·수혜/
  제출·보고/트랙 구성/제도적 특징)하여 KG 개념이 없는 정책 전문가·정치인도
  주요 이해관계와 지원의 성격을 읽을 수 있게 했습니다.
- BK21은 **미래인재 트랙(파랑) / 혁신인재 트랙(주황)** 으로 노드를 구분하여,
  단순한 대학 지원이 아니라 분야별·트랙별 지원이라는 특징을 살렸습니다.
- 노드에 마우스를 올리면 이웃 관계만 강조되고 설명이 표시됩니다. 노드는 드래그 가능.

### ② 시뮬레이션 (원탁 토의)
- 다섯 명의 이해관계자(평가위원·정책담당자·박사후연구원)가 **원탁에 둘러앉아**
  단계별(IAD 논리모형: 투입→활동→산출→성과→영향) 결론과 근거를 말풍선으로 제시합니다.
- **Forward → Backward** 진행만 보여줍니다(우리 실험에서 macro-avg가 가장 좋았던 구성).
- 각 단계는 `개별 추정(이상치 노출) → 토의 후 합의(이상치 흡수·수렴) →
  역방향 재검토 → 교차검증`의 흐름으로, **이상치가 사라지고 합의에 이르는 과정**과
  **forward/backward가 어긋나 합의가 깨진 지점**을 함께 드러냅니다.
- 오른쪽 패널은 변수별 forward·backward 추정값과 합의도(consistency)를 시각화합니다.

### ③ 전문가 피드백
- 일반적 정책 평가가 아니라, **이 정책 성격(R&D·인재양성)에 맞는 실제 평가기준** 위에
  피드백을 배치했습니다.
  - **국가연구개발 성과평가 체계**(과기정통부·KISTEP, 자체평가–점검–메타평가 3단계)
  - **정책 고유 성과지표**(BK21 붙임3: 취업률·전공일치율·전임교수 강의비율, SSF 단계평가·지역 30%)
  - **OECD-DAC 6대 평가기준**(적절성·일관성·효과성·효율성·영향·지속가능성)
- 각 항목은 *어떤 기준의 어떤 세부 지표*에서 제기되는 지적인지, *어떤 전문가*(정책전문가·
  사회학자·정치학자·정치인·현장실무자)가, *정책 구조/시뮬레이션 중 무엇*에 대해
  말하는지를 함께 표시해 객관성과 체계성을 높였습니다.

## 데이터 파이프라인

- `web/build_simulation.py` — 원본 `scenario_*.json` 로그에서 forward→backward 구간만
  추출해 프런트엔드용 `web/data/simulation.json`을 생성합니다.
- `web/data/graph.json` — 정책 문서 기반으로 재구성한 이해관계 그래프(BK21·SSF).
- `web/data/feedback.json` — 평가기준별 전문가 피드백 항목.

```bash
python3 web/build_simulation.py   # simulation.json 재생성
```
